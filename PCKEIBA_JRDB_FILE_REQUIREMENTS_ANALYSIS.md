# JRDB 提供ファイル 必要性分析 (Phase 6予測用)

## 📋 概要

JRDBから提供される15種類のファイルのうち、**Phase 6 当日予測システムに必要なのは4ファイルのみ**。残り11ファイルは登録をスキップし、処理時間を短縮する。

---

## ✅ 必須ファイル (4種類) - Phase 6予測に使用

### 1. **KYI (競走馬)**
- **ファイル名例**: `KYI260221.txt`
- **内容**: 競走馬の基本情報・IDM・予想情報・馬体重・オッズ等
- **Phase 6での用途**: 
  - 馬の基本属性 (年齢、性別、毛色等)
  - IDM (Index for Derby Match) - JRDB独自の総合指標
  - 展開予想・上がり予想
  - 馬体重・増減
  - 前走情報
- **重要度**: ⭐⭐⭐⭐⭐ (Phase 6の最重要データソース)
- **Phase 1特徴量との対応**:
  ```python
  # extract_jra_features_v1.py での利用箇所
  jrd_kyi.idm                  # JRDB総合指標
  jrd_kyi.bataiju             # 馬体重
  jrd_kyi.zogen               # 馬体重増減
  jrd_kyi.tenkaikubun         # 展開予想 (逃げ・先行・差し等)
  jrd_kyi.agari_yoso          # 上がり予想
  ```

### 2. **CYB (調教)**
- **ファイル名例**: `CYB260221.txt`
- **内容**: 調教タイム・調教コース・調教量・調教指数等
- **Phase 6での用途**:
  - 最終追い切りタイム
  - 調教コース種別 (栗東・美浦・坂路・プール等)
  - 調教量 (本馬場・坂路の本数)
  - 調教評価 (助手・厩舎コメント)
- **重要度**: ⭐⭐⭐⭐⭐ (調教状態は予測精度に直結)
- **Phase 1特徴量との対応**:
  ```python
  jrd_cyb.cyoshisu            # 調教指数
  jrd_cyb.oikiri_time         # 追い切りタイム
  jrd_cyb.cyokyo_course       # 調教コース
  jrd_cyb.cyokyo_ryou         # 調教量
  ```

### 3. **JOA (情報)**
- **ファイル名例**: `JOA260221.txt`
- **内容**: 調教師・騎手コメント、注目馬情報、レース展開予想等
- **Phase 6での用途**:
  - 厩舎コメント (陣営の意気込み・仕上がり評価)
  - 騎手コメント (乗り方・展開予想)
  - 注目度ランク
  - 推奨買い目情報
- **重要度**: ⭐⭐⭐⭐ (コメント系特徴量の抽出に必要)
- **Phase 1特徴量との対応**:
  ```python
  jrd_joa.cyoshi_comment      # 調教師コメント
  jrd_joa.kisyu_comment       # 騎手コメント
  jrd_joa.chumoku_rank        # 注目度ランク
  ```

### 4. **SED (成績) - 結果確定後のみ**
- **ファイル名例**: `SED260221.txt`
- **内容**: 確定着順・タイム・通過順位・上がり3ハロン・賞金等
- **Phase 6での用途**: 
  - **レース前の予測では不要** (結果が未確定のため)
  - **レース後のモデル再学習** (Phase 3～5) で使用
  - Phase 6予測の検証 (予測 vs. 実績の比較)
- **重要度**: ⭐⭐⭐ (予測時は不要、事後検証・再学習で必須)
- **備考**: 金曜夜のダウンロード時点では土曜・日曜レースの SED は存在しない

## ❌ 不要ファイル (11種類) - Phase 6登録をスキップ可能

以下のファイルは、Phase 6当日予測では使用しないため、週次登録から除外することで処理時間を大幅短縮できる。

### 5. **BAC (馬毎レース前情報)**
- **内容**: 過去走の詳細分析・ペース指数・展開記号等
- **Phase 6での利用**: なし (Phase 1～2の特徴量抽出で一部利用可能だが、KYIで代替可能)

### 6. **CHA (馬毎調教分析)**
- **内容**: 週間調教の詳細分析・調教パターン
- **Phase 6での利用**: なし (CYBで代替可能)

### 7. **KAB (馬別成績)**
- **内容**: 過去の全成績・コース別成績・騎手別成績等
- **Phase 6での利用**: なし (Phase 1でJRA-VAN jvd_seから抽出済み)

### 8. **OZ (オッズ情報)**
- **内容**: 単勝・複勝・馬連・3連複等の詳細オッズ
- **Phase 6での利用**: なし (KYIに主要オッズが含まれる)

### 9. **OW (ワイド)**
- **内容**: ワイドオッズの詳細
- **Phase 6での利用**: なし

### 10. **OU (馬単)**
- **内容**: 馬単オッズの詳細
- **Phase 6での利用**: なし

### 11. **OT (3連単)**
- **内容**: 3連単オッズの詳細
- **Phase 6での利用**: なし

### 12. **UKC (馬券攻略)**
- **内容**: JRDB推奨の買い目・点数・配当予想
- **Phase 6での利用**: なし (当システムは独自のLightGBMモデルで予測)

### 13. **KKA (馬毎過去走分析)**
- **内容**: 過去10走の詳細ペース分析
- **Phase 6での利用**: なし (KABで代替可能)

### 14. **ZKB (前日情報)**
- **内容**: 前日の馬体重・オッズ・出走取消情報
- **Phase 6での利用**: なし (KYIに含まれる)

### 15. **ZED (前日成績)**
- **内容**: 前日の結果速報
- **Phase 6での利用**: なし (SEDで代替可能)

## 📊 処理時間への影響 (推定)

| 設定パターン | 登録ファイル種類 | 処理ファイル数 | 推定所要時間 |
|:-----------|:---------------|:-------------|:-----------|
| **現状 (全ファイル)** | 15種類 | 15種類 × 15年分 = 約225ファイル | **2時間30分** ⚠️ |
| **最適化後 (4ファイル)** | KYI/CYB/JOA/SED のみ | 4種類 × 直近30日分 = 約12ファイル | **3～5分** ✅ |
| **削減率** | -73% | -95% | **-97%** |

## ⚙️ PC-KEIBA設定での除外方法

### 方法A: 設定ファイル (XML) でファイル種別を限定

`C:\Users\<username>\Documents\PC-KEIBA\config\jrdb_config.xml` (パスは要確認) を編集:

```xml
<FileTypes>
  <!-- Phase 6予測に必須の4ファイルのみ有効化 -->
  <File enabled="true">KYI</File>
  <File enabled="true">CYB</File>
  <File enabled="true">JOA</File>
  <File enabled="true">SED</File>
  
  <!-- 以下11ファイルは登録スキップ -->
  <File enabled="false">BAC</File>
  <File enabled="false">CHA</File>
  <File enabled="false">KAB</File>
  <File enabled="false">OZ</File>
  <File enabled="false">OW</File>
  <File enabled="false">OU</File>
  <File enabled="false">OT</File>
  <File enabled="false">UKC</File>
  <File enabled="false">KKA</File>
  <File enabled="false">ZKB</File>
  <File enabled="false">ZED</File>
</FileTypes>
```

### 方法B: データフォルダの分離

週次運用では、必要な4ファイルのみをダウンロード＆配置:

```
C:\JRDB\phase6\          ← 週次運用用 (PC-KEIBAの登録対象フォルダに指定)
  ├─ KYI260221.txt      ✅
  ├─ CYB260221.txt      ✅
  ├─ JOA260221.txt      ✅
  └─ SED260221.txt      ✅ (レース後のみ)

C:\JRDB\archive\         ← 過去データ保管用 (登録対象外)
  ├─ BAC*.txt
  ├─ CHA*.txt
  └─ ... (その他11種類)
```

## ✅ 結論

### 週次運用での推奨設定

1. **初回セットアップ (1回のみ)**:
   - 全15ファイル × 2016～2025年分を一括登録 (所要時間: 2～3時間)
   - この作業は Phase 1～5 のモデル開発時に完了済み

2. **毎週の運用 (金曜夜～土曜朝)**:
   - **KYI, CYB, JOA の3ファイルのみ**をダウンロード＆登録
   - 登録対象期間: 直近30日 (または当日のみ)
   - 差分登録モード: ON
   - 所要時間: **3～5分以内** ← **目標達成**

3. **レース後の事後処理 (土曜夜・日曜夜)**:
   - **SED (成績)** をダウンロード＆登録
   - 予測精度検証スクリプトを実行
   - 必要に応じて Phase 3～5 のモデル再学習 (月次でまとめて実施)
