# PC-KEIBA JRDBデータ登録設定 確認・最適化指示書

## 🎯 **確認目的**

**現状の問題**:
- データ登録を押すと、毎回15年分のJRDBデータを処理している
- 1回の登録に膨大な時間がかかる（数時間？）
- Phase 6で必要なのは**当日（または直近）のデータのみ**

**理想の状態**:
- 当日分（または直近1週間分）のJRDBデータのみを登録
- 1回の登録時間を5分以内に短縮
- 過去データは一度だけ登録し、以降は新規データのみ追加

---

### ✅ **Phase 6で本当に必要なJRDBデータの範囲**

# scripts/phase6/phase6_daily_prediction.py の該当箇所

- **Phase 6実行時には、過去データを再読み込みする必要はない**

## 🔧 **最適化の方針**

### **方針A: 差分登録（増分登録）に変更（推奨）**

**概要**:
- 初回のみ過去データを登録（1回だけ時間がかかる）
- 以降は新規データ（当日分）のみを追加登録
- PostgreSQLには全期間のデータが蓄積される

**設定方法**:
```
1. PC-KEIBAの「JRDB設定」を開く
2. 以下の設定を変更:
   □ 「差分登録」または「増分登録」を ON にする
   □ 「全データ再登録」を OFF にする
   □ 「既存データを削除しない」を選択

3. 初回のみ、過去データを一括登録:
   - 「登録対象期間」を「2016-2025」に設定
   - 「データ登録」を実行（1-2時間かかる）
   - PostgreSQLに全データが蓄積される

4. 以降、毎週の運用では:
   - 「登録対象期間」を「直近30日」または「自動（新規のみ）」に変更
   - データ登録は数分で完了
```

**メリット**:
- 毎週の登録時間が劇的に短縮（5分以内）
- PostgreSQLに全期間のデータが保持される
- Phase 1-5の再学習時にも使用可能

---

### **方針B: 当日分のみ手動登録（シンプル）**

**概要**:
- 過去データは一切登録しない
- 毎週、当日分（または直近1週間分）のみを手動で登録
- PostgreSQLには直近のデータのみが保持される

**設定方法**:
```
1. PC-KEIBAの「JRDB設定」を開く
2. 「登録対象期間」を「手動指定」に設定
3. 毎週の運用時:
   - 開始日: 2026-02-22（当日）
   - 終了日: 2026-02-22（当日）
   - データ登録を実行（1-2分で完了）
```

**メリット**:
- 登録時間が最短（1-2分）
- データベースの容量が最小

**デメリット**:
- Phase 1-5の再学習時に過去データが使えない
- データ分析時に過去データが参照できない

---

### **方針C: 当日分のみ自動追加（最も理想的）**

**概要**:
- PC-KEIBAが自動的に新規データを検出
- 既存データは保持したまま、新規データのみを追加
- ユーザーは「データ登録」ボタンを押すだけ

**設定方法**:
```
1. PC-KEIBAの「JRDB設定」を開く
2. 「自動増分登録」または「スマート登録」を ON にする
3. 「新規データのみ追加」を選択
4. 以降、「データ登録」を押すだけで当日分が自動追加される
```

**メリット**:
- 最も手間がかからない
- 登録時間が最短
- PostgreSQLに全データが蓄積される
