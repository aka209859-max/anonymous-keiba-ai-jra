## 🔧 予測データ生成方法

### スクリプト: `batch_predict_2025_monthly.py`

#### 機能
- 2025年1月～12月の全開催日の予測を**月別**に生成
- データ量が多いため、月ごとに分割実行可能
- 各開催日ごとに `scripts/phase6/phase6_daily_prediction.py` を実行

#### 実行方法

```powershell
cd E:\anonymous-keiba-ai-JRA

# スクリプト取得
git pull origin genspark_ai_developer

# 実行
python batch_predict_2025_monthly.py
```

#### 実行オプション

実行時に選択：
- **0**: 全月（1月～12月）を一括実行
- **1～12**: 特定の月のみ実行

**推奨**: まず **1月のみ** でテスト実行し、問題なければ全月実行

#### 推定実行時間

| 実行範囲 | 開催日数 | 推定時間 |
|---------|---------|---------|
| 1月のみ | 約20日 | 10～30分 |
| 全月（1～12月） | 約220～260日 | **2～4時間** |

#### 出力ファイル

各開催日ごとに以下のファイルが生成される：

```
results/predictions_20250105.csv
results/predictions_20250112.csv
results/predictions_20250119.csv
...
results/predictions_20251228.csv
```

**合計ファイル数**: 約220～260個  
**合計サイズ**: 約200～500MB

## 🚀 次のステップ

### ステップ1: 予測データ生成 🔄 進行中

```powershell
cd E:\anonymous-keiba-ai-JRA
python batch_predict_2025_monthly.py
```

**推奨実行手順**:
1. まず「1」を選択して1月のみ実行（テスト）
2. 問題なければ「0」を選択して全月実行

### ステップ2: Phase 2C 実装 ⏳ 待機中

予測CSV生成完了後、以下を実装：

1. **データ読み込み・マージ**
   - 予測CSV読み込み
   - jvd_o1 からオッズをパース
   - jvd_se/jvd_hr から着順・払戻を取得
   - 全データを結合

2. **偏差値計算**
   - レースごとに `ensemble_score` を標準化
   - 偏差値 = 50 + 10 × (score - mean) / std
   - S/A/B/C/D/E ランクに分類

3. **マトリクス集計**
   - 偏差値ランク × オッズ帯でグルーピング
   - 件数、的中数、的中率、回収率、ROI を計算

4. **結果出力**
   - CSV マトリクス
   - ヒートマップ画像
   - Markdown レポート

## 📞 問題発生時の対応

### よくある問題

#### 1. ModuleNotFoundError
```
ModuleNotFoundError: No module named 'lightgbm'
```
**対処**: `pip install lightgbm pandas numpy pyyaml psycopg2`

#### 2. データベース接続エラー
```
psycopg2.OperationalError: FATAL: password authentication failed
```
**対処**: `config/db_config.yaml` のパスワードを確認

#### 3. モデルファイルが見つからない
```
FileNotFoundError: models/binary_model.txt
```
**対処**: モデルファイルの存在を確認、必要に応じて再トレーニング
