### **Phase 7-B: ファクターROI厳選戦略**

#### 基準の明確化

**生の回収率100%を基準とする理由**:
- 補正なし = 実運用に即した評価
- 100%超 = 利益が出る条件
- 100%未満 = 単体では赤字（組み合わせで救済）

#### 評価方法

```python
def calculate_factor_roi(df, factor_name):
    """
    ファクター単体の生ROI計算
    
    例: 「騎手勝率」で上位20%の馬だけ購入した場合
    """
    threshold = df[factor_name].quantile(0.8)
    selected = df[df[factor_name] >= threshold]
    
    # 生ROI計算（補正なし）
    hit_rate = selected['is_top3'].mean()
    avg_odds = selected['tansho_odds'].mean()
    roi = (hit_rate * avg_odds) * 100  # %表記
    
    return {
        'factor': factor_name,
        'roi': roi,
        'baseline_score': roi,  # 100を基準としたスコア
        'hit_rate': hit_rate,
        'avg_odds': avg_odds
    }
```

#### ランク分類

| ランク | ROI範囲 | 判定 | 想定数 |
|--------|---------|------|--------|
| **S** | ≥ 110% | 超優良 | 10~20個 |
| **A** | 100~110% | 優良 | 20~40個 |
| **B** | 90~100% | 平均的 | 50~80個 |
| **C** | < 90% | 単体弱い | 50~100個 |

**重要**: Cランクも削除せず、組み合わせで救済

### **Phase 7-G: ROI特化モデル構築戦略**

#### LightGBMカスタム損失関数

```python
def roi_loss(y_pred, dtrain):
    """
    期待ROIを直接最適化する損失関数
    
    loss = -(予測勝率 × 前日オッズ - 1)
    """
    y_true = dtrain.get_label()
    odds = dtrain.get_weight()  # 前日オッズ
    
    # 期待値ベース勾配
    ev = y_pred * (odds - 1) - (1 - y_pred)
    grad = -odds * (y_true - y_pred) + (1 - y_true)
    hess = odds * y_pred * (1 - y_pred) + (1 - y_pred)
    
    return grad, hess
```

#### アンサンブル設計

| モデル | 役割 | 重み |
|--------|------|------|
| **LightGBM ROI特化** | 期待値直接最適化 | 60% |
| **Benter統合** | 市場知恵の取り込み | 40% |

**最終予測**:
```
最終勝率 = 0.6 × LightGBM予測 + 0.4 × Benter統合予測
期待ROI = 最終勝率 × 前日オッズ × 100
```

### **Phase 7-H: バックテスト戦略**

#### 資金管理方式の比較

**方式A: PAT残高×8%（卍氏方式）**
```
賭け金 = 現在資金 × 8% ÷ 前日オッズ

メリット: シンプル、実装容易
デメリ: 期待値を考慮しない
```

**方式B: Fractional Kelly（係数0.25）**
```
賭け金 = 資金 × (勝率×オッズ - 1) / (オッズ - 1) × 0.25

メリット: 期待値とリスクのバランス最適
デメリ: 勝率推定誤差に敏感
```

#### 評価指標

| 指標 | 計算式 | 目標値 |
|------|--------|--------|
| **年間ROI** | (最終資産 - 初期資産) / 総賭け金 × 100 | ≥ 105% |
| **Sharpe Ratio** | (平均収益 - 無リスク金利) / 収益標準偏差 | ≥ 1.5 |
| **最大ドローダウン** | max(資産ピーク - 現在資産) / 資産ピーク | < 30% |
