### 目的

### CrossFactor方式の3つの補正

#### 1. 期間別の重み付け補正
**問題点**: 古いデータ（2016年）と新しいデータ（2025年）を同じレベルで扱っている。

**解決策**: 新しいデータほど重みを大きくする。
```python
year_weights = {
    2016: 1.0,
    2017: 2.0,
    2018: 3.0,
    2019: 4.0,
    2020: 5.0,
    2021: 6.0,
    2022: 7.0,
    2023: 8.0,
    2024: 9.0,
    2025: 10.0
}
```

**理由**:
- 騎手の騎乗能力の年齢による変化
- 馬券術や競馬予想AIの進化によるオッズの変動
- より新しいデータの方が現在の競馬環境を反映

#### 2. オッズ補正係数の適用
**問題点**: オッズによる回収率の偏りを考慮していない。

**実証データ（2016-2020年）**:
- **単勝**: オッズ100倍以上 → 回収率が一気に下がる
- **複勝**: オッズ17倍以上 → 回収率が一気に下がる

**解決策**: オッズ範囲ごとに補正係数を適用。
```python
# 単勝オッズ補正係数
tansho_correction = {
    (1.0, 1.9):   1.00,  # 基準
    (2.0, 2.9):   1.02,
    (3.0, 4.9):   1.05,
    (5.0, 9.9):   1.08,
    (10.0, 19.9): 1.10,
    (20.0, 49.9): 1.05,
    (50.0, 99.9): 0.90,
    (100.0, inf): 0.50  # 大幅減衰
}

# 複勝オッズ補正係数
fukusho_correction = {
    (1.0, 1.4):   1.00,  # 基準
    (1.5, 1.9):   1.02,
    (2.0, 2.9):   1.05,
    (3.0, 4.9):   1.08,
    (5.0, 9.9):   1.10,
    (10.0, 16.9): 1.05,
    (17.0, inf):  0.60  # 大幅減衰
}
```

#### 3. 均等払戻による分析（最重要！）

**従来の「均等買い」の問題**:
```
対象データ数: 100件
的中件数: 1件
配当: 15,000円

→ 回収率: 150%
```
**問題**: この1件の的中がなければ回収率は0%。穴馬の影響が大きすぎる。

**CrossFactorの「均等払戻」**:
- すべての馬券を**同じ払戻金額（例: 10,000円）**で買った場合の回収率
- オッズが低い馬券 → 多くの金額をベット
- オッズが高い馬券 → 少ない金額をベット

**計算式**:
```python
# 均等買い（従来）
bet_amount = 100  # 固定
recovery_rate = sum(payoff) / sum(bet_amount)

# 均等払戻（CrossFactor）
target_payoff = 10000  # 目標払戻額（固定）
bet_amount = target_payoff / odds  # オッズに応じて変動
recovery_rate = sum(actual_payoff) / sum(bet_amount)
```

**効果**: 穴馬券の影響度を下げ、より安定した回収率分析が可能。

#### 4. 信頼度の計算

CrossFactorの信頼度計算式:
```
信頼度 = (補正回収率 - 基準値80) × データ件数
```

**解釈**:
- 信頼度が高いほど、データの信憑性が高い
- プラスの信頼度 → 期待値が高い
- マイナスの信頼度 → 期待値が低い

### 実装内容

#### 2.1 オッズフィルター設計

**オッズ帯の定義**:
- 1.0-1.4倍: 除外（過度な本命、回収率低下リスク）
- 1.5-2.9倍: 低リスク本命帯
- 3.0-5.9倍: バランス型
- 6.0-9.9倍: 中穴狙い
- 10.0倍以上: 大穴狙い（超高リスク）

**注意事項**:
- 当日予測時はリアルタイムオッズを取得しない
- 確定オッズとの差異について注意書きを記載
- オッズフィルターは過去データ検証用

#### 2.2 検証パターン

**パターン1: 偏差値1位 × 単勝オッズ帯**
- 偏差値≥70 × 単勝1.0-1.9倍
- 偏差値≥70 × 単勝2.0-2.9倍
- 偏差値≥70 × 単勝3.0-5.9倍
- 偏差値≥70 × 単勝6.0-9.9倍
- 偏差値≥70 × 単勝10.0倍以上

（偏差値65-69.9、60-64.9についても同様）

**パターン2: 偏差値2-5位 × 複勝オッズ帯**
- 偏差値≥65 × 複勝1.0-1.1倍
- 偏差値≥65 × 複勝1.2-1.4倍
- 偏差値≥65 × 複勝1.5-1.9倍
- 偏差値≥65 × 複勝2.0-2.9倍
- 偏差値≥65 × 複勝3.0倍以上

（偏差値60-64.9、55-59.9についても同様）

#### 2.3 検証指標

各パターンで以下を集計:
- **対象レース数**: そのパターンに該当したレース数
- **的中数**: 実際に的中した回数
- **的中率**: 的中数 / 対象レース数
- **平均配当**: 的中時の平均払戻金額
- **回収率**: (総払戻金額 / 総購入金額) × 100
- **ROI**: 回収率 - 100 (%)
- **推奨度**: ★★★★★（ROI +10%以上）〜 ★（ROI -20%以下）

## 📈 Phase 3: 偏差値化 + ランク評価

### 目的
各レース内で相対評価を行い、偏差値を計算してランク付けする。Phase 2の検証結果を基に、購入推奨ロジックを組み込む。

### 実装内容

#### 3.1 偏差値計算

**計算式**:
```python
# 各レース内で相対評価
mean = ensemble_score.mean()
std = ensemble_score.std()
z_score = (ensemble_score - mean) / std
hensachi = 50 + 10 * z_score  # 平均50、標準偏差10の偏差値
```

**特殊ケース処理**:
- 標準偏差が0の場合（全馬同スコア）: 全馬偏差値50
- 出走頭数が2頭の場合: 偏差値計算はするが推奨度を下げる

#### 3.2 ランク定義

| 偏差値範囲 | ランク | 評価 |
|------------|--------|------|
| 70.0以上 | S | 最有力（超本命） |
| 65.0-69.9 | A | 有力（本命候補） |
| 60.0-64.9 | B | 注目（対抗候補） |
| 55.0-59.9 | C | 妙味あり（穴候補） |
| 50.0-54.9 | D | 評価普通 |
| 49.9以下 | E | 評価低い |

#### 3.3 購入推奨ロジック

**レベル1: レース全体の推奨度**
- 偏差値1位馬の偏差値でレース全体を評価
- 例: 「🏇 阪神 第1R 予想 【購入推奨: S】」

**レベル2: 各馬の推奨度**
- 偏差値ランク（S/A/B/C/D/E）
- オッズ妙味評価（Phase 2の検証結果を反映）

**推奨ロジック（Phase 2検証後に確定）**:
```python
if 1位馬_偏差値 >= 70 and 1位馬_オッズ < 1.5:
    # 低オッズ本命は除外
    if 2位馬_偏差値 >= 65 and 2位馬_オッズ >= 3.0:
        推奨 = "2位馬（偏差値Aランク、オッズ妙味あり）"

    推奨 = "1位馬（偏差値Sランク）"

    # 偏差値とオッズのバランスを考慮
    推奨 = determine_by_validation_result()
```

## 🧩 背景と課題認識

### 現状の問題点

1. **アンサンブルスコアの絶対評価**
   - 現在のスコア（0.72、0.71など）は各馬の絶対的な勝率予測
   - レース間でスコア分布が異なるため、単純比較が難しい
   - 例: あるレースの0.72が「強い」のか、別のレースの0.68が「弱い」のか不明

2. **購入推奨の基準が不明確**
   - 現在は単純にスコア上位3頭を推奨
   - オッズを考慮していないため、回収率が低い可能性
   - 例: 単勝1.1倍の馬が偏差値1位でも回収率は低い

3. **オッズ妙味の評価不足**
   - 偏差値2位の馬が単勝5.5倍、偏差値1位が1.3倍の場合
   - 継続的に購入すべきは2位の馬（妙味がある）
   - しかし現状は1位馬を推奨してしまう

### 解決策

#### Phase 2 で解決すること:
- オッズ帯別の実証的な回収率データを取得
- どのオッズ帯が最も効率的かを特定
- 3つの戦略を比較し、最適な購入ロジックを確立

#### Phase 3 で解決すること:
- 各レース内で相対評価（偏差値化）
- オッズと偏差値を組み合わせた推奨ロジック
- 回収率を最大化する購入推奨を提示
