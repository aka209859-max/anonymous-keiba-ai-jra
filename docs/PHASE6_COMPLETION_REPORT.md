### **2. `kaisai_nichime` 16進数パース問題**

#### **問題**
323件目以降で `value too long for type character varying(1)` エラー

#### **原因**
```python
# 誤ったコード
nichi_hex = hex_to_int(value)  # 'a' → 10
record['kaisai_nichime'] = str(nichi_hex)  # '10' (2文字)
```

- `kaisai_nichime` (日目) が16進数1バイト
  - `'8'` = 8日目
  - `'a'` = 10日目（16進数）
- スクリプトが10進数に変換して2文字化
- PostgreSQL のカラム型が `VARCHAR(1)` で格納不可

#### **解決策**
```python
# 修正後
record['kaisai_nichime'] = value  # 'a' → 'a' (1文字のまま)
```

16進数文字をそのまま格納することで、VARCHAR(1) 制限を回避

**結果**:
- 497件全てインポート成功（0件スキップ）
- エラーなし

### **3. Phase 6 の `race_shikonen` クエリ不一致**

#### **問題**
Phase 6 で `✅ JRDB特徴量マージ: 0件` と表示

#### **原因**
```sql
-- 誤ったクエリ
WHERE '20' || kyi.race_shikonen = %s           -- パラメータ: '2026'
  AND kyi.keibajo_code = ANY(%s)
  AND SUBSTRING(kyi.race_shikonen, 1, 4) = %s  -- パラメータ: '2602'
```

**問題点**:
- `'20' || '260222' = '20260222'` を `'2026'` と比較 → 不一致
- `race_shikonen` は6桁 (`260222`) で格納されている
- WHERE句が常にFALSEになり、0件取得

#### **解決策**
```sql
-- 修正後のクエリ
WHERE kyi.race_shikonen = %s  -- パラメータ: '260222'
  AND kyi.keibajo_code = ANY(%s)
```

**結果**:
```
✅ JRDB特徴量マージ: 497件 (対象日: 0222)
```

## 📝 **技術的ドキュメント**

### **JRDB ファイルフォーマット**

**KYI (騎手指数) - 1024 bytes/record**

| 位置 | 長さ | 項目 | 例 |
|------|------|------|-----|
| 1-2 | 2 | 競馬場コード | `05` |
| 3-4 | 2 | 年（YY） | `26` |
| 5 | 1 | 回 | `1` |
| 6 | 1 | 日（16進） | `8` または `a` |
| 7-8 | 2 | レース番号 | `01` |
| 9-10 | 2 | 馬番 | `01` |
| 11-18 | 8 | 血統登録番号 | `23103049` |
| 19-54 | 36 | 馬名 | `ネオアンボウ` |
| 55-59 | 5 | IDM | `10.0` |
| ... | ... | ... | ... |

**重要**: `kaisai_nichime` (日目) は16進数1文字
- `'1'` = 1日目
- `'9'` = 9日目
- `'a'` = 10日目
- `'b'` = 11日目
- `'c'` = 12日目
